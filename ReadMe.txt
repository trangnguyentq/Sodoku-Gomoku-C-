B. Explanation of Our Design and Copy Control Design in this lab:

1.We didn't use any copy/move/copy-assignment/move-assignment constructors and destructors because we think that the original game object will be destructed if the user chooses to quit.

In that case, we will not be able to pass in the original game object to the copy constructor to contruct a same game.

So we decided not to use copy constructor. Instead, we chose to store all the information needed to restore the game in the file with the name of the game

and to contruct a new one if the user chooses to save the game and restarts playing.

2. The ways we store the information for different game are:

(1)Gomoku.txt format:

	<player> <x-coordinate> <y-coordinate>

	e.g:

	X 1 1

	O 2 2

(2)TicTacToe.txt format:

	<player> <x-coordinate> <y-coordinate>

	e.g:

	B 1 1

	W 2 2

(3)Sudoku.txt format:

	<y-coordinate> <x-coordinate> <tile>

	e.g:

	0 0 1
